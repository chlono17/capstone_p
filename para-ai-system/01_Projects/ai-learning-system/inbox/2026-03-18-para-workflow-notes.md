# PARA 工作流筆記

## 核心想法

敏捷開發型學習者需要以「行動」為核心的系統，而不是以「收集」為核心。

傳統筆記系統的問題：
- 花太多時間整理，沒有時間執行
- 筆記變成倉庫，不是工具
- 缺乏「行動觸發」機制

## 解決方案：Git-Native PARA

把所有筆記和代碼放在同一個 Git 倉庫。這樣：
1. 每個 commit 就是一個學習里程碑
2. AI 可以透過 git log 追蹤進度
3. 開發和學習紀錄統一管理

## n8n 自動化的關鍵洞察

手寫筆記 → 英文文章 → 播客，這個流程解決了兩個問題：
1. 降低語言障礙：用母語思考，自動轉換成英文輸出
2. 多模態複習：看文字 + 聽音頻，強化記憶留存率
