# CLAUDE.md — para-ai-system Project Conventions

## Project Identity
- **Purpose**: Agile personal knowledge management + AI automation pipeline
- **Source notes**: Any language (Chinese/English/mixed)
- **Output articles**: English
- **Output podcasts**: MP3 audio

## Directory Conventions
| Directory | Purpose |
|-----------|---------|
| `01_Projects/` | Active work with a deadline or deliverable. One subdirectory per project. |
| `02_Areas/` | Ongoing responsibilities with no end date (e.g., `team-standards/`, `ai-prompts/`). |
| `03_Resources/` | Reference material, templates, research notes, paper summaries. |
| `backlog/` | Parked ideas, deferred tasks, inbox. Review weekly. |

## File Naming Rules
- All Markdown notes: `YYYY-MM-DD-slug.md`
- No spaces in filenames; use hyphens.
- English articles generated by n8n: `YYYY-MM-DD-slug-article.md`
- Podcast files: `YYYY-MM-DD-slug-podcast.mp3`

## n8n Automation Pipeline
- **Trigger**: Drop a `.md` file into `01_Projects/<project-name>/inbox/`
- **Step 1**: Claude API converts notes (any language) → polished English article
- **Step 2**: TTS API converts article → MP3 podcast
- **Output**: Files saved beside the source note (outside `inbox/`)

## Slash Commands
- `/summary <file>` — Structured summary of any Markdown note
- `/todo` — Scan all PARA directories and surface actionable items

## Linking Convention
- Use `[[Page Title]]` double-bracket links for cross-note references (Obsidian-compatible)
- External links: standard Markdown `[text](url)`

## Code Style (scripts)
- Shell scripts: `#!/usr/bin/env bash` + `set -euo pipefail`
- No hardcoded API keys — always read from environment variables
