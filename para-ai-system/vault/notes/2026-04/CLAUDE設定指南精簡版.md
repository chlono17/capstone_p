## 摘要
1. 分析 CLAUDE.md 长度，限制在 200 行内。
2. 利用 Memory 记录特定项目的长期进展和短期状态，不用手动输入。
3. 在日常工作中随时更新 CLAUDE.md 规则。
4. 使用 rules 文件管理特定路径的条件性规则。
5. 设置 hooks 来强制执行高风险操作中的逻辑。

**Q: 你对 CLAUDE.md 的长度限制有什么具体想法，或者是如何有效地利用 Memory 记录重要的临时状态？**
[具體答案 2-4 句，必須引用原始內容的細節]
 尽量保持 CLAUDE.md 在 200 行以内；内存记录临时状态的步骤：输入 `/memory` 记录内容，使用 `{/memory}` 查看和编辑。

**Q: 动态更新 CLAUDE.md：当你需要添加新的规则或修改现有的规则时，你会如何确保这些更改能够被有效地记录和应用到 CLAUDE.md 中？**
[具體答案 2-4 句，必須引用原始內容的細節]

 当检测到模型要发送特定文件或自动保存进度更改时，可以设置 hooks 来强制执行这些规则。
